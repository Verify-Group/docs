---
title: Authentification
description: Authentifiez les requêtes de l'API Verify Group avec des jetons JWT bearer, renouvelez le jeton d'accès et définissez l'en-tête d'organisation multi-tenant requis.
---

Chaque requête doit inclure `Authorization: Bearer <token>`.

## Connexion

```bash
POST /api/v1/auth/login
```
```json
{"email":"user@example.com","password":"yourpassword"}
```

**Réponse :**
```json
{"data":{"accessToken":"eyJ...","refreshToken":"eyJ...","expiresIn":3600}}
```

## Renouveler un jeton

```bash
POST /api/v1/auth/refresh
```
```json
{"refreshToken":"eyJ..."}
```

## En-têtes requis

| En-tête | Requis | Description |
|---|---|---|
| `Authorization` | Toujours | `Bearer <token>` |
| `X-Organization-ID` | La plupart des points de terminaison | UUID de votre organisation |

## Bonnes pratiques de sécurité

<Check>Stockez les jetons d'accès en mémoire, pas dans localStorage</Check>
<Check>Stockez les jetons de rafraîchissement dans des cookies sécurisés httpOnly</Check>
<Check>Utilisez HTTPS partout</Check>
<Check>Révoquez les jetons lors de la déconnexion</Check>
