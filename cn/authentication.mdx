---
title: 身份认证
description: 使用 JWT bearer token 对 Verify Group API 请求进行认证，刷新访问令牌，并设置多租户所需的组织请求头。
---

每个请求都必须包含 `Authorization: Bearer <token>`。

## 登录

```bash
POST /api/v1/auth/login
```
```json
{"email":"user@example.com","password":"yourpassword"}
```

**响应：**
```json
{"data":{"accessToken":"eyJ...","refreshToken":"eyJ...","expiresIn":3600}}
```

## 刷新令牌

```bash
POST /api/v1/auth/refresh
```
```json
{"refreshToken":"eyJ..."}
```

## 必需的请求头

| 请求头 | 是否必需 | 描述 |
|---|---|---|
| `Authorization` | 始终必需 | `Bearer <token>` |
| `X-Organization-ID` | 大多数接口 | 你所在组织的 UUID |

## 安全最佳实践

<Check>将访问令牌保存在内存中，而不是 localStorage</Check>
<Check>将刷新令牌保存在 httpOnly 的安全 cookie 中</Check>
<Check>全程使用 HTTPS</Check>
<Check>用户登出时吊销令牌</Check>
