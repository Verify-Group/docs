---
title: 证据管理
description: 通过代理接口上传理赔证据、为文件打标签、执行审核审批，并管理证据在 Verify Group API 中的状态流转。
---

## 上传（推荐浏览器端使用）

将上传通过后端进行中转，以避免 CORS 问题：

```bash
POST /api/v1/evidence/public-upload/proxy-upload
Content-Type: multipart/form-data
```

返回 `evidenceId` 以及可立即用于预览的 `signedUrl`。

## 列出某个理赔的证据

```bash
GET /api/v1/evidence?claimId={claimId}
```

## 为证据打标签

```bash
PATCH /api/v1/evidence/{id}/tags
```
```json
{"tags": ["police_report", "verified", "high_priority"]}
```

## 审批或驳回证据

```bash
POST /api/v1/evidence/{id}/review
```
```json
{"status": "APPROVED", "notes": "Document authenticated against police database."}
```

| 状态 | 含义 |
|---|---|
| `PENDING` | 等待审核 |
| `APPROVED` | 已核验通过 |
| `REJECTED` | 已驳回 |
| `UNDER_REVIEW` | 鉴证分析中 |
