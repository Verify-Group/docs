---
title: Autenticação
description: Autentique requisições da API Verify Group com tokens JWT bearer, atualize o token de acesso e defina o cabeçalho de organização multi-tenant obrigatório.
---

Toda requisição deve incluir `Authorization: Bearer <token>`.

## Login

```bash
POST /api/v1/auth/login
```
```json
{"email":"user@example.com","password":"yourpassword"}
```

**Resposta:**
```json
{"data":{"accessToken":"eyJ...","refreshToken":"eyJ...","expiresIn":3600}}
```

## Atualizar um token

```bash
POST /api/v1/auth/refresh
```
```json
{"refreshToken":"eyJ..."}
```

## Cabeçalhos obrigatórios

| Cabeçalho | Obrigatório | Descrição |
|---|---|---|
| `Authorization` | Sempre | `Bearer <token>` |
| `X-Organization-ID` | Maioria dos endpoints | UUID da sua organização |

## Boas práticas de segurança

<Check>Armazene tokens de acesso em memória, não em localStorage</Check>
<Check>Armazene refresh tokens em cookies httpOnly seguros</Check>
<Check>Use HTTPS em todos os lugares</Check>
<Check>Revogue tokens no logout</Check>
